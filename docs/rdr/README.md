# palinex RDRs

Architectural decisions for the palinex project. Lightweight convention — markdown files only, no T2 lifecycle, no automated gates. Each RDR captures one durable decision with enough context that a future reader can judge whether the decision still holds.

Numbering: `rdr-NNN-kebab-title.md`, sequential. Frontmatter mirrors the nexus RDR convention (title, id, type, status, priority, author, created, related_rdrs).

Status values: `draft`, `accepted`, `superseded`, `withdrawn`.

When updating an RDR: never rewrite history. Append a `## Revision History` entry, or mark status and write a successor.
